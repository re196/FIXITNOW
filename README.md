FixITNow

Objective

The FixITNow project was developed to improve campus infrastructure management by creating a transparent and efficient issue-reporting platform for students and staff. The application allows users to report problems such as damaged classroom equipment, electrical issues, water leakage, cleanliness concerns, and other maintenance-related complaints directly through a mobile-based system. The main objective was to simplify communication between students and the management while ensuring faster issue resolution and better tracking of maintenance activities.

This project provided practical experience in application design, problem-solving, teamwork, and understanding how technology can be used to solve real-world campus management problems.

Challenges Faced During Development

Although the FixITNow project successfully passed the Zeroth Review, First Review, and Second Review stages, several technical issues appeared during the final implementation phase. The project was primarily developed using Lovable, which helped in rapid UI creation and workflow management. However, during the final review, some core functionalities and integrations did not work properly as expected.

The major challenges included:

System instability during final testing.

Backend and feature integration issues.

Errors in workflow execution and data handling.

Difficulty in maintaining smooth functionality across all modules.

Limited time for debugging and optimization before the final review.


Due to these technical difficulties, the project could not be completed to the expected final working condition, and development was temporarily stopped. Despite this, the project provided valuable learning experience in project planning, UI/UX development, teamwork, testing, debugging, and understanding real-world software development challenges.
